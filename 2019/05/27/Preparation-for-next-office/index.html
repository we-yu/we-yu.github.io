What is CLOUD

  • クラウド(クラウドコンピューティング)とは…
    • ユーザ側が物理的なマシンやアプリケーション・ソフトウェアを持たなくてもサービスを必要な時に必要な分だけ(オンデマンド)受けられる考え方
    • ユーザはネットワーク経由でコンピューティングサービスを受け取る。
    • 対義語はオンプレミス(On-premises)。
      • クラウド型と比較しての「従来の」サービス提供形態。情報システムのハードウェア・ソフトウェアを物理的に自身の管轄下に置いて運用する。
    • もっと一般的な喩えにするなら、ピザ屋の配達のようなもの。（具体的な店舗や調理にかんしてユーザは知る必要がない）

利点/欠点

  • (オンプレミス)と比べて導入・運用が容易であること。
    • 基本的に既存サービスを導入する形になるため、習熟的にも初期コストが低い。
    • ハードウェアの導入・維持管理費も含めて全てサービス提供側が負うため運用面のコストが低い。また、ソフトウェアのバージョンも自動的に更新される。
    • 月額／１ユーザ額等料金プランによるがオンデマンドに使用できる。
    • 必要なサービスだけ選択して導入することが可能。
  • クラウド型では初期投資・保守のコストがおさえられるが、反応速度・セキュリティ・カスタマイズ性では劣る(場合がある)。また従量課金制の場合はコストが予測しきれない。
    • 特に超ハイレベルのセキュリティが求められる金融・軍事面ではオンプレミスの方が選ばれる傾向がある。
    • 顧客ごとへの柔軟なカスタマイズ対応が苦手。特定の顧客に紐付いた大規模請負フルスクラッチ開発のようにはいかない。
      • ただしどう考えても開発者には利点。

パブリッククラウド／プライベートクラウド

  • セキュリティやカスタマイズ性を希求する場合は、クラウドサービスを公開ネットワーク経由で享受するパブリッククラウドとは別に、自社サーバーを中心としたプライベートクラウド環境を構築するという手がある。
    • ただしこの場合は構築維持管理費が発生するため、クラウドのメリットを幾分か捨てている形になる。
  • ハードウェアを自身で持つ必要がないため、オンプレミスからフルクラウドへ移行する傾向にある。
  • 一例として、従来のOutlookによるメール管理がオンプレミス、Gmailがクラウドサービス。

クラウドサービスの種類

  • Saas, Paas, Iaas. 各々(Software/Platform/Infrastructure) as a Service.
  • SaaS : ソフトウェア提供型. ※Gmail
  • IaaS : 開発環境提供型 ※Docker
  • IaaS : サーバ(インフラ)提供型 ※ALTUS, AWS

AWS

  • サーバのみならずユーザの声を反映した多種多様なサービスを持っている。
  • とりあえずLinuxサーバ運用のみに絞るならEC2が対象になる。
